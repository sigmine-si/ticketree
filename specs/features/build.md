# 개발과 배포

버전: v1.0 · 마지막 변경 2026-07-22

## 이렇게 동작해요

- [x] 명세가 승인되면 개발이 시작되고, 명세에 적힌 예정 항목이 그대로 수용 기준이 된다
- [x] 개발은 다른 요청과 격리된 작업 공간에서 이뤄져 서로 방해하지 않는다
- [x] 작업 결과는 검토 가능한 형태로 남고, 관리자 승인 전에는 반영되지 않는다
- [x] 개발 중에 명세만으로 판단할 수 없는 문제를 만나면 멈추고 관리자에게 넘긴다
- [x] 그렇게 멈춘 경우에도 그때까지 한 작업은 버리지 않고 남긴다
- [x] 관리자가 답을 주면 처음부터가 아니라 멈춘 지점부터 이어서 진행한다
- [x] 관리자가 배포를 승인하면 변경이 반영된다
- [x] 배포가 끝나면 요청이 종결되고 명세의 예정 항목이 정식 항목이 된다
- [x] 클라이언트에게는 접수됨 → 개발 중 → 개발 완료 → 배포 완료 네 단계만 보인다

## 알려진 제약

- 실제 배포는 사람이 직접 한다. 플랫폼은 반영까지만 하고 "배포 완료"는 관리자가 표시한다
- 미리보기 주소도 관리자가 직접 입력한다
- 자동 되돌리기는 없다
